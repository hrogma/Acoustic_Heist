# 🎭 Acoustic Heist

**Стелс-игра на Python**, где главное оружие — тишина. Проникните в здание, соберите монеты и доберитесь до выхода, избегая глаз и ушей бдительных охранников.

## 🎯 Особенности

- 🔇 **Акустическая механика** — шум от шагов и бега распространяется по карте, привлекая охранников
- 👁️ **Реалистичное поле зрения** — враги видят только то, что не скрыто за стенами (Shadowcasting)
- 🧠 **Умный AI** — охранники патрулируют, расследуют шум и преследуют нарушителя (FSM + A*)
- 🏗️ **Чистая архитектура** — ECS, EventBus, разделение Model/View/Controller
- 📐 **Следование принципам** — SOLID, DRY, KISS

### Принципы архитектуры

| Принцип | Как реализован |
|---------|----------------|
| **Single Responsibility** | Каждая система отвечает только за свою задачу |
| **Open/Closed** | Новые враги добавляются через компоненты, без изменения систем |
| **Dependency Inversion** | Системы зависят от абстракций (компонентов), а не друг от друга |
| **DRY** | Общая логика вынесена в `domain/` и переиспользуется |
| **KISS** | Простые алгоритмы вместо избыточных решений |
| **MVC** | Model (`domain/`), View (`presentation/`), Controller (`systems/`) |

## 🧠 Алгоритмы в игре

### 1. Shadowcasting (Поле зрения)
Реалистичное поле зрения с учётом стен. Враг видит только те клетки, до которых нет препятствий. Поддерживается конус зрения (угол обзора).

```python
fov = compute_fov(origin_x, origin_y, radius, grid, facing_angle, fov_angle)
```

### 2. A* Pathfinding (Поиск пути)
Враги ищут кратчайший путь к цели, обходя стены. Используется эвристика Манхэттена.

```python
path = find_path(grid, start_x, start_y, goal_x, goal_y)
```

### 3. Dijkstra Map (Распространение звука)
Шум от игрока распространяется по карте с затуханием, огибая стены. Враги слышат звук и идут к его источнику.

```python
sound_map = compute_sound_map(grid, source_x, source_y, intensity, decay)
```

### 4. FSM (Конечный автомат)
AI врагов переключается между состояниями:

```
PATROL ──(шум/визуал)──▶ INVESTIGATE ──(нашёл)──▶ CHASE
   ▲                                                   │
   └──────────(потерял, таймаут)── SEARCH ◀────────────┘
```

### 5. Bresenham's Line (Прямая видимость)
Проверка, видит ли враг игрока напрямую, без препятствий на линии.

---

## 🎨 Состояния AI врагов

| Состояние | Цвет | Поведение |
|-----------|------|-----------|
| `PATROL` | 🔵 Синий | Движется по заданным точкам |
| `INVESTIGATE` | 🟡 Жёлтый | Идёт к источнику шума |
| `CHASE` | 🔴 Красный | Преследует игрока |
| `SEARCH` | 🟣 Фиолетовый | Ищет игрока в области |
| `ALERT` | 🟠 Оранжевый | Реагирует на обнаружение |

```bash

### Добавление нового уровня

1. Откройте `levels/level1.py`
2. Создайте новый файл `levels/level2.py` по аналогии
3. Используйте символы:
   - `#` — стена
   - `.` — пол
   - `P` — игрок
   - `E` — враг
   - `C` — монета
   - `X` — выход
   - `N` — источник шума

### Добавление нового типа врага

```python
# Создайте врага с особыми параметрами
enemy.add(PerceptionComponent(
    vision_range=4,          # Короткий радиус зрения
    vision_angle=180.0,      # Широкий угол
    hearing_sensitivity=0.05 # Очень чуткий слух
))
```
